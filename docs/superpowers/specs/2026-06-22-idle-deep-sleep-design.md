# 输入空闲 + 摄像头确认的深度休眠 — 设计文档（桌面版）

## 背景与目标

v1.5.3 已有"动态检测"：姿态稳定（人不动）时把检测间隔从 5s 自适应拉长到 20s，
坏姿势/离开立即恢复密集检测。但它只看**摄像头里的姿态稳定度**，没用上**电脑输入
活动**（键鼠）。

真实场景：人离开工位时，电脑既无键鼠操作、画面里也没人。此时还每 5-20s 开一次
摄像头探测纯属浪费 CPU/电，也和视频通话争用摄像头。

**目标**：当摄像头判定无人、且键鼠长时间空闲（人确实不在）时，进入"深度休眠"——
**完全关闭摄像头**、只极低成本地轮询键鼠空闲；一旦碰键鼠立刻恢复正常检测。

**关键约束（用户明确）**：
1. 不另起独立的空闲计时状态机；输入信号**结合**现有摄像头判定——进入由摄像头
   away 主导，逻辑长在现有 away 分支里。
2. **退出纯靠键鼠唤醒，不做摄像头兜底**。理由：主动使用电脑必然触发键鼠输入；
   "在座但不碰键鼠"（看视频/读屏）属被动放松场景，不监测姿势可接受。

## 核心机制

### 系统空闲读取

新增极轻量的系统空闲查询，封装在独立函数里：

```
CGEventSourceSecondsSinceLastEventType(
    kCGEventSourceStateHIDSystemState, kCGAnyInputEventType)
```

返回自上次任意键鼠输入以来的秒数。纯系统调用、无 I/O、零成本。

- macOS 专有（依赖 Quartz）。取不到（非 mac / API 异常）→ 返回 `None`，调用方
  视为"不空闲"，整套深度休眠**退化为现有行为**，不影响其它平台。

### 三态状态机

| 状态 | 条件 | 行为 |
|------|------|------|
| **正常检测** | 画面有人 | 现有逻辑：5→20s 动态间隔 |
| **away** | 画面无人（现有 away 检测） | 现有逻辑不变：暂停媒体、提醒等 |
| **深度休眠（新）** | 处于 away **且** 键鼠空闲 ≥ 300s | 关摄像头；每 ~2s 轻量轮询空闲秒数；碰键鼠即唤醒 |

### 进入深度休眠

同时满足：
1. 主循环当前判定为 **away**（画面连续无人，沿用现有 away 判定）；
2. 键鼠空闲秒数 ≥ **300s（5 分钟）**；
3. **非 AC 供电**（电池供电）。深度休眠是省电功能，插电时无电池压力，应保持
   正常监控、不为省电牺牲姿势检测。读不到电源（None）当作电池处理，不禁用功能。
   注意：台式 Mac 恒为 AC → 永不进入深度休眠（本期接受此后果）。

第 2 条的"且空闲"是**防摄像头误判**的与门：若用户正在打字、摄像头却因光线/遮挡
误报无人，输入不空闲就不会进深度休眠。

### 深度休眠期间

- 摄像头**关闭**（不为任何检测重开）；
- 主循环以 ~2s 周期只做轻量轮询：读键鼠空闲秒数，判断是否该唤醒；
- 期间不开摄像头、不检测姿势、不喂统计。

### 退出深度休眠

- **唯一触发**：键鼠活动——空闲秒数 < `WAKE_IDLE`(5s)，说明刚有键鼠输入、人回来了。
- 唤醒后：开摄像头、沿用现有"丢帧预热"避免冷启动误判，恢复正常检测的密集间隔。
- failsafe：休眠中若空闲秒数变得取不到（`None`）也唤醒，回退到现有行为。

## 单元划分与边界

- **新增纯逻辑模块**（可单测，不依赖真摄像头/真键鼠）：
  - `read_input_idle_seconds() -> float | None`：读系统空闲秒数，失败回 `None`。
  - `read_on_ac_power() -> bool | None`：读电源（`pmset -g batt` 解析 AC/Battery），失败回 `None`。
  - 决策函数 `deep_sleep_decision(in_deep_sleep, is_away, idle_seconds, on_ac_power=None) -> action`：
    输入当前状态 + away + 空闲秒数 + 电源，输出动作（`enter` / `wake` / `stay_sleep` / `none`）。
    `on_ac_power=True` 时未休眠一律 `none`（不进入）；唤醒逻辑不受电源影响。
    常量：`ENTER_IDLE_SECONDS=300`、`WAKE_IDLE_SECONDS=5`、`DEEP_SLEEP_POLL_SECONDS=2`。
- **`core.py`（改）**：在主循环顶部接入决策函数——休眠中按动作轮询/唤醒；现有 away
  分支里触发进入。
- 深度休眠期**不喂 `progression`**（away 本就不计良好率，行为一致）。
- 菜单栏状态：休眠期沿用现有 **away/人不在** 显示，不新增图标。
- 视频通话：摄像头本就让出，休眠期不抢，一致。

## 错误处理 / 边界

| 情况 | 处理 |
|------|------|
| 非 mac / Quartz 缺失 / API 异常 | `read_input_idle_seconds()` 回 `None` → 不进深度休眠；休眠中遇 `None` 则唤醒，退回现有行为 |
| 摄像头误报无人但用户在打字 | 输入不空闲（< 300s）→ 不进深度休眠 |
| 人回来只看屏不碰键鼠（看视频/读屏） | 设计上接受：此被动场景不监测姿势，直到碰键鼠唤醒 |
| 唤醒后冷启动暗帧 | 沿用现有丢帧预热 |
| `self.running` 置否（停止监控） | 轮询用现有可中断 `_sleep`，及时退出 |

## 参数（默认值）

- 进入深度休眠的键鼠空闲门槛：**300s（5 分钟）**，且必须同时 away
- 判定"键鼠活动"的空闲阈值（唤醒）：**5s**
- 休眠期轮询周期：**2s**

（如需可后续提为 `Settings` 字段；本期先用模块常量，YAGNI。）

## 测试

- **单元**（决策函数 + 空闲读取）：
  - 未休眠 + away + 空闲≥300 → `enter`；
  - 未休眠 + 空闲<300 → `none`；非 away → `none`；`idle=None` → `none`；
  - 休眠中空闲<5 → `wake`；休眠中 `idle=None` → `wake`(failsafe)；
  - 休眠中仍空闲(≥5) → `stay_sleep`。
- **集成手测**：离开工位 5 分钟 → 摄像头停（指示灯灭）；碰鼠标 → 秒级恢复。

## 数据收集与评估（backtest）

目的：跑一段时间后量化深度休眠**是否有效**——既看"省了多少"，也看"误进入"。
评估是**观测式**：埋足够的事件，事后用脚本聚合。

### 埋点（新增 `deepsleep_events.jsonl`，与 posture.jsonl 同目录、同 RotatingFileHandler 风格）

| 事件 | 字段 |
|------|------|
| `deep_sleep_enter` | `ts`, `idle_seconds` |
| `deep_sleep_exit` | `ts`, `duration_s`（本次休眠时长） |
| `away_idle_snapshot` | 摄像头刚判定 away 那一刻记一次：`ts`, `idle_seconds`（用于事后调 ENTER_IDLE 门槛） |

摄像头检测频率无需单独埋——每次检测本就写 posture.jsonl，**按小时计数 posture.jsonl
条数**即可反推检测密度（含休眠前后对比）。

### 评估脚本（`scripts/eval_deep_sleep.py`，纯离线读日志）

读 `deepsleep_events.jsonl` + `posture.jsonl`，输出报告：

- **省电侧**：累计休眠时长、会话数、时长中位/均值；休眠前后 **posture 条数/小时** 对比
  （反推摄像头/检测密度下降幅度）。
- **误进入侧**：进入后很快（<30s）就退出的会话占比——值高说明睡早了；用
  `away_idle_snapshot` 复算不同门槛（60/120/300s）下的进入时机，辅助调 ENTER_IDLE。

脚本只读日志、不依赖运行中进程，可随时跑（"过一段时间"后执行即可）。
